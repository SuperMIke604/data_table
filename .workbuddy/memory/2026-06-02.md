# 2026-06-02 data_table 工作日志

## 修复：报错日志复制按钮 PC 端无反应
- 问题：PC 端网页版点击"复制"和"复制全部"按钮无反应，手机端正常
- 根因：
  1. `navigator.clipboard.writeText()` 在非 HTTPS（非安全上下文）下直接抛异常
  2. 单条复制的 fallback 用了 `document` 而非 `this.ownerDocument`，独立窗口/iframe 场景下 DOM 操作挂错文档
- 修复：
  - 新增通用函数 `copyTextToClipboard(text, doc)`：优先 Clipboard API（仅安全上下文），降级到 `textarea + execCommand('copy')`
  - 单条复制按钮：改为 `copyTextToClipboard(text, this.ownerDocument)`，不再用 `navigator.clipboard`
  - "复制全部"按钮：同样改为 `copyTextToClipboard(text, doc)`
  - fallback 的 textarea 增加了 `focus()` + `top:-9999px;opacity:0;` 确保可靠选中
